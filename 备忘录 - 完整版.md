### 🐛 Bug 修复：资讯文章缺少 _key 问题

**问题描述**：
自动化生成的资讯文章在 Sanity Studio 中显示 "Missing keys" 错误，无法编辑。

**根本原因**：
`publisher.ts` 中构建 content 字段时，block 和 span 元素缺少必需的 `_key` 字段。

**解决方案**：
已修复 `scripts/news-auto/publisher.ts`，为所有语言版本的 content 添加 `_key`：
```typescript
content: {
  zh: [
    {
      _key: 'zh_block_1',      // ✅ 新增
      _type: 'block',
      children: [
        {
          _key: 'zh_span_1',   // ✅ 新增
          _type: 'span',
          text: processed.content.zh,
        },
      ],
    },
  ],
  // ... 其他语言版本 (en/id/th/vi/ar) 同样处理
}
```

**影响范围**：
- ✅ **已修复**：未来自动发布的文章将不再出现此问题
- ⚠️ **需手动**：现有 5 篇文章已在 Studio 中手动修复完成

**下次任务执行时间**：明日凌晨自动采集时将使用新代码，不会再出现此问题。

### 浏览器语言自动检测 ✅ (2026 年 3 月 12 日完成)

**实现功能**：根据用户浏览器 `Accept-Language` 头自动跳转到对应语言版本

**技术实现**：
- 创建 `middleware.ts` 中间件检测浏览器语言
- 支持语言：zh-CN/zh → /zh, id/ms → /id, th → /th, vi → /vi, ar → /ar, 其他 → /en
- 使用 302 临时重定向，禁用缓存避免问题

**问题排查过程**：
1. 最初尝试使用 `app/page.tsx` 服务端组件实现，但存在静态生成冲突
2. 发现 `next.config.mjs` 中有硬编码重定向规则 `/` → `/en`
3. 发现 `vercel.json` 中有边缘网络层重定向规则（根本原因！）
4. 删除所有硬编码重定向，统一由中间件处理

**关键教训**：
- Vercel 配置优先级：`vercel.json` > `next.config.mjs` > 中间件
- 永久重定向（308）会被浏览器和 CDN 长期缓存，开发时应使用临时重定向（302）
